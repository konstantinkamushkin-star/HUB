# Руководство по отладке координат

## ✅ Что проверено

1. **База данных:** Все 712 дайвсайтов имеют валидные координаты
2. **API:** Возвращает координаты в правильном формате `{"latitude": X, "longitude": Y}`
3. **PostGIS:** Координаты соответствуют location::geometry

## 🔍 Как проверить проблему

### 1. Проверьте логи в Xcode Console

После запуска приложения ищите в консоли:
- `📍 [DiveSite] Decoded:` - показывает декодированные координаты
- `📍 [DiveSite] Location created:` - показывает созданные координаты
- `🗺️ [MapViewModel] Creating annotation:` - показывает координаты аннотаций
- `🗺️ [MapViewModel] Centering map:` - показывает центр карты

### 2. Проверьте ответ API

```bash
curl "http://localhost:3000/api/v1/dive-sites/search?lat=27.5&lng=34.0&radius=100000&limit=3"
```

Проверьте, что:
- `latitude` находится в диапазоне [-90, 90]
- `longitude` находится в диапазоне [-180, 180]
- Для Египта: `latitude` ≈ 27-28, `longitude` ≈ 33-35

### 3. Проверьте координаты в БД

```sql
SELECT name, latitude, longitude, country 
FROM dive_sites 
WHERE country = 'Egypt' 
LIMIT 5;
```

### 4. Проверьте отображение на карте

- Откройте карту в приложении
- Проверьте, видны ли аннотации
- Если не видны, проверьте начальную позицию карты
- Попробуйте переместить карту вручную

## ⚠️ Возможные проблемы

1. **Координаты не видны на карте:**
   - Начальная позиция карты: Карибское море (20.0, -80.0)
   - Дайвсайты могут быть в других регионах
   - Решение: карта теперь автоматически центрируется на загруженных дайвсайтах

2. **Координаты перепутаны:**
   - Проверьте логи - там будет видно, если координаты перепутаны
   - API возвращает правильно, проблема может быть в декодировании

3. **Аннотации не отображаются:**
   - Проверьте, что `annotations` не пустой массив
   - Проверьте, что координаты валидны в логах
