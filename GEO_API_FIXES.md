# Исправления проблем с геопоиском

## Проблемы, которые были исправлены

### 1. ✅ Локация не обновляла фильтры автоматически
**Проблема**: Локация пользователя получалась, но не передавалась в фильтры для геопоиска.

**Решение**:
- Добавлен `LocationManager` в `ExploreViewModel`
- Автоматическое обновление `filters.centerLatitude` и `filters.centerLongitude` при получении локации
- Debouncing обновлений локации (1 секунда) для оптимизации

### 2. ✅ Нет обработки ошибок при недоступности нового API
**Проблема**: Если новый geo API endpoint не существует (404), приложение падало.

**Решение**:
- Добавлена обработка ошибок с fallback на старый API
- Специальная обработка 404 ошибки (endpoint не найден)
- Детальное логирование для отладки

### 3. ✅ Кэш показывал устаревшие данные
**Проблема**: Кэш не очищался при изменении фильтров.

**Решение**:
- Автоматическая очистка истекших записей кэша перед загрузкой
- Debouncing обновлений локации для предотвращения частых перезагрузок

### 4. ✅ Недостаточное логирование
**Проблема**: Сложно было понять, что происходит при загрузке данных.

**Решение**:
- Добавлено детальное логирование в DEBUG режиме
- Логи показывают: используется ли геопоиск, есть ли локация, какой API используется

## Как проверить, что все работает

### 1. Проверка получения локации
В консоли должны появиться логи:
```
📍 [ExploreViewModel] Location updated in filters: 20.0, -80.0
🔍 [ExploreViewModel] Loading data - Location: ✅, Geo search: ✅
```

### 2. Проверка использования геопоиска
Если новый API доступен:
```
📍 [ExploreViewModel] Geo search successful: 25 sites
```

Если новый API недоступен (404):
```
⚠️ [ExploreViewModel] Geo search API endpoint not found (404), using legacy API
```

### 3. Проверка кэша
```
📍 [ExploreViewModel] Using cached geo search results: 25 sites
```

## Отладка

### Если локация не определяется:
1. Проверьте разрешения в `Info.plist`:
   - `NSLocationWhenInUseUsageDescription`
   - `NSLocationAlwaysAndWhenInUseUsageDescription`

2. Проверьте логи:
   - Должно быть: `Location: ✅`
   - Если `Location: ❌` - проверьте разрешения

### Если геопоиск не работает:
1. Проверьте, что backend поддерживает новый API:
   - Endpoint: `/api/v1/dive-sites/search`
   - Если нет - будет использован старый API автоматически

2. Проверьте логи:
   - `Geo search: ✅` - геопоиск должен использоваться
   - `Geo search: ❌` - проверьте, что `centerLatitude` и `centerLongitude` установлены

### Если показываются старые данные:
1. Очистите кэш:
   ```swift
   GeoCacheService.shared.clearCache()
   ```

2. Проверьте TTL кэша (по умолчанию 5 минут)
