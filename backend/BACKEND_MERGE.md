# Объединение старого и нового бэкенда

## Обзор

Объединена функциональность старого Go бэкенда (`backend_examples/go_service/`) с новым NestJS бэкендом (`backend/`).

## Что было добавлено из Go бэкенда

### 1. Кластеризация дайвсайтов
- **Endpoint**: `GET /api/v1/dive-sites/clusters`
- **Параметры**:
  - `north`, `south`, `east`, `west` - границы области поиска
  - `zoom` - уровень масштаба карты (1-20)
  - `difficulty` - фильтр по сложности
  - `site_types` - фильтр по типам сайтов
  - `min_rating` - минимальный рейтинг
- **Функциональность**: Группирует дайвсайты в кластеры на основе уровня масштаба карты
- **Реализация**: `DiveSitesService.getClusters()`

### 2. Улучшенный Health Check
- **Endpoint**: `GET /api/health`
- **Проверки**:
  - Статус базы данных (PostgreSQL)
  - Статус кэша (Redis или in-memory)
- **Ответ**: JSON с статусом каждого компонента
- **Реализация**: `AppController.health()`

### 3. Performance Tracking Middleware
- **Функциональность**: Добавляет заголовок `X-Response-Time` к каждому ответу
- **Реализация**: `PerformanceInterceptor`
- **Использование**: Глобальный interceptor для всех запросов

### 4. Улучшенный CORS
- Добавлены все необходимые заголовки и методы
- Поддержка credentials

## Структура API

### Основные endpoints

1. **Поиск по местоположению**
   - `GET /api/v1/dive-sites/search?lat={lat}&lng={lng}&radius={radius}`
   - Поддержка фильтров, сортировки, пагинации

2. **Поиск по границам (для карты)**
   - `GET /api/v1/dive-sites/map?north={north}&south={south}&east={east}&west={west}`
   - Оптимизирован для отображения на карте

3. **Популярные дайвсайты**
   - `GET /api/v1/dive-sites/popular?country={country}&limit={limit}`
   - Fallback когда нет местоположения

4. **Кластеризация** (новое)
   - `GET /api/v1/dive-sites/clusters?north={north}&south={south}&east={east}&west={west}&zoom={zoom}`
   - Возвращает кластеры и отдельные точки в зависимости от уровня масштаба

5. **Health Check** (улучшен)
   - `GET /api/health`
   - Проверяет состояние БД и кэша

## Технические детали

### Кластеризация

Алгоритм кластеризации работает следующим образом:
- **Высокий zoom (>15)**: Возвращает все точки без кластеризации
- **Средний zoom (10-15)**: Смешанный режим - кластеры и отдельные точки
- **Низкий zoom (<10)**: Только кластеры

Расстояние для кластеризации: ~1км (0.01 градуса)

### Кэширование

- Redis кэш с fallback на in-memory
- TTL зависит от радиуса поиска:
  - < 10км: 5 минут
  - < 50км: 15 минут
  - >= 50км: 1 час

### Производительность

- Все запросы отслеживаются через `X-Response-Time` заголовок
- Оптимизированные PostGIS запросы с индексами
- Кэширование результатов поиска
