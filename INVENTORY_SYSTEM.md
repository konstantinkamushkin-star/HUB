# Система управления оборудованием (Inventory Management System)

## Обзор

Реализована комплексная система управления оборудованием для дайв-центра с полным функционалом отслеживания, обслуживания и инвентаризации.

## Структура проекта

### Модели данных

1. **GearItem** (`Models/Gear.swift`)
   - Расширенная модель с полями: серийный номер, штрихкод/QR, локация, ответственный, даты проверок, состояние, фото, документы
   - Состояния: available, issued, maintenance, lost, retired, scrapped
   - Условия: new, good, needs_service, damaged, lost, retired

2. **InventoryModels** (`Models/InventoryModels.swift`)
   - `Location` - локации/склады
   - `Inspection` - проверки оборудования с чек-листами
   - `ChecklistTemplate` - шаблоны чек-листов
   - `Checkout` - выдача/возврат оборудования
   - `MaintenanceTicket` - заявки на обслуживание
   - `InventoryAudit` - инвентаризация
   - `EventLog` - история событий
   - `InventoryKPIs` - метрики и статистика
   - `WarningItem` - предупреждения

### ViewModels

- **InventoryViewModel** (`ViewModels/InventoryViewModel.swift`)
  - Управление всеми данными инвентаря
  - Фильтрация, поиск, сортировка
  - Группировка по категориям
  - Массовые операции
  - Расчет KPI и предупреждений

### Views

1. **InventoryDashboardView** - Главный дашборд
   - KPI карточки (всего единиц, доступно, на обслуживании, выдано, просрочено)
   - Список предупреждений
   - Быстрые действия
   - Графики трендов

2. **InventoryListView** - Список оборудования
   - Три режима отображения: список, группировка, карточки
   - Поиск по ID, названию, серийному номеру, штрихкоду
   - Расширенные фильтры (категория, статус, состояние, локация)
   - Сортировка
   - Множественный выбор для массовых действий

3. **ItemDetailView** - Детальная карточка единицы
   - Вкладки: Основное, История, Обслуживание, Брони, Документы, Инспекции, Связанные элементы
   - Полная информация об оборудовании
   - Хронология событий
   - История обслуживания и выдач

4. **AddEditItemView** - Добавление/редактирование
   - Все поля оборудования
   - Валидация (уникальность серийного номера)
   - Шаблоны (готово к расширению)
   - Теги и категории

5. **CheckoutView** - Выдача/возврат
   - Многошаговый процесс
   - Выбор получателя
   - Фиксация состояния при выдаче
   - Подпись получателя

6. **InspectionView** - Проверка оборудования
   - Шаблоны чек-листов
   - Пошаговое прохождение
   - Фото к пунктам
   - Результат: passed/failed/conditional
   - Автоматический расчет следующей проверки

7. **MaintenanceTicketsView** - Заявки на обслуживание
   - Список заявок с фильтрами
   - Приоритеты и статусы
   - Детальная информация
   - Запчасти и стоимость

8. **ReportsView** - Отчёты
   - Использование по периоду
   - Стоимость ремонтов
   - Соответствие инспекциям
   - ROI анализ
   - Экспорт CSV/PDF

9. **InventoryTabView** - Главная навигация модуля

## Интеграция с существующим кодом

Система интегрирована с:
- `StorageService` - для локального хранения
- `NetworkService` - для API вызовов (готово к подключению)
- `LocalizationService` - для локализации
- `AuthenticationService` - для определения текущего пользователя

```swift

    InventoryTabView()
}

// Или отдельные экраны

    InventoryDashboardView()
}
```

## Примечания

- Все модели поддерживают Codable для сериализации
- Используется soft delete для истории
- Поддержка множественных локаций/складов
- Готово к расширению для мультиязычности
- Структура готова для добавления ролей и прав доступа
