## 🔍 Как проверить, что координаты правильные

### 1. Проверьте логи в Xcode Console

После запуска приложения ищите:
```
📍 [DiveSite] Decoded: Ras Mohammed - lat=27.7333, lng=34.2833, country=Egypt
📍 [DiveSite] Location created: Ras Mohammed - coordinate lat=27.7333, lng=34.2833
🗺️ [MapViewModel] Creating annotation: Ras Mohammed
   Location: lat=27.7333, lng=34.2833
   Coordinate: lat=27.7333, lng=34.2833
```

**Если видите несоответствие** - координаты перепутаны при декодировании или создании.

### 2. Проверьте на карте

- Откройте карту в приложении
- Дайвсайты должны автоматически центрироваться
- Для Египта координаты должны быть около 27-28°N, 33-35°E
- Если дайвсайты не видны, проверьте начальную позицию карты

### 3. Проверьте API напрямую

```bash
curl "http://localhost:3000/api/v1/dive-sites/search?lat=27.5&lng=34.0&radius=100000&limit=3"
```

Проверьте формат ответа:
```json
{
  "latitude": 27.7333,  // ← это широта (latitude)
  "longitude": 34.2833  // ← это долгота (longitude)
}
```

## ⚠️ Если проблема сохраняется

2. **Проверьте, что backend запущен** - `http://localhost:3000`
3. **Пересоберите приложение** - изменения в Swift требуют пересборки
4. **Очистите кэш** - Product → Clean Build Folder в Xcode

## 📝 Важные моменты

- **API формат:** `{"latitude": X, "longitude": Y}` - правильный порядок
- **Swift декодирование:** `latitude` → `location.latitude`, `longitude` → `location.longitude`
- **CLLocationCoordinate2D:** `CLLocationCoordinate2D(latitude: lat, longitude: lng)` - правильный порядок
- **MapKit:** Использует координаты напрямую из CLLocationCoordinate2D
